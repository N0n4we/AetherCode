# NAT 带宽优化

## 背景

成本分析发现 NAT 网关 7 月 1-13 日已产生 ¥111，是最大开销源，但 TKE VPC-CNI 集群必须保留 NAT 作为唯一出公网通道。通过降带宽优化。

## 网络拓扑

```
VPC 10.0.0.0/16
└── 路由 0.0.0.0/0 → NAT nat-6hvu0buf (EIP 1.15.179.177)
```

NAT 是 TKE Pod 访问上游 API（OpenAI/Anthropic 等）的唯一出口，不可删除。

## 变更

**NAT 网关 EIP 带宽降配：20Mbps → 1Mbps**

- 资源：`eip-1q4hutfv`（绑定 `nat-6hvu0buf`）
- Terraform：`platform/terraform/tencentcloud/nat.tf`
- 变更字段：`tencentcloud_eip.aether_nat.internet_max_bandwidth_out`

## 效果

| 指标 | 变更前 | 变更后 |
|------|--------|--------|
| 带宽 | 20Mbps | 1Mbps |
| 月费估算 | ~¥256 | ~¥13 |
| 月节省 | - | ~¥243 |

## 风险与回滚

1Mbps 足够 relay API 请求，但大 payload 或高并发可能成为瓶颈。

回滚：改回 `internet_max_bandwidth_out = 20`，`terraform apply` 即时生效。
