# 成本优化提示

把资源占用、空闲证据和成本事实整理成**可验证、可执行**的优化建议；不写正式报告、背景综述或目录。

## 优先级

1. **可执行性**：能否自主验证和执行。
2. **空闲证据**：是否明确闲置、可回收。
3. **利用率**：是否长期低利用。
4. **成本金额**：只用于估算收益，不作为第一排序依据。

## 每条建议保留

- **对象**：产品、实例 ID/名称、业务归属。
- **依据**：利用率、空闲状态、API 证据；说明时间窗口。
- **动作**：释放、合并、降配、转计费方式、调整保留期等。
- **影响**：风险、回滚难度、是否需低峰窗口。
- **节省**：能给数字就给；不能确定则给区间或验证条件。

## 输出格式

- 用短清单或小表格。
- 只保留决策所需信息。

## 安全边界

- 没有数据支撑，不写确定结论。
- 只有账单高、没有利用率或空闲证据时，只给验证步骤。
- 释放、删除、降配、切流、改公网入口等动作，执行前必须让用户确认。
- 高风险建议优先给验证步骤，不直接给破坏性命令。
