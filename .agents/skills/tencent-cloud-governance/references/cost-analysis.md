# 成本分析提示

目标是回答“钱花在哪、为什么、是否合理”。必须使用cli查询实时状态。

## 常见切入点

- 总量：余额、月度总额、Top 产品。
- 趋势：最近几个完整月，找异常上升或长期固定的大头。
- 结构：按产品、付费方式、地域、资源实例。
- 归因：把账单里的实例 ID / 资源名与资源 API 或本地快照对上。
- 浪费信号：长期按量、低流量 CLB、闲置 EIP/CBS、过期快照、规格明显偏大的数据库/缓存/计算集群、标签缺失导致归属不清。

## 输出要求

- 说清楚数据窗口和数据来源。
- 金额带单位；余额注意分/元换算。
- 区分事实、推断和需要补充监控验证的点。
- 如果给优化建议，要说明风险和是否需要用户确认。
